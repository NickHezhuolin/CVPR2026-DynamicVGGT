# Contributing to DynamicVGGT

We welcome pull requests that improve code quality, documentation, and reproducibility.

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. Keep changes focused and match existing code style.
3. Update documentation when APIs or configs change.
4. Do not commit private checkpoints, dataset paths, or credentials.

## Issues

Please include steps to reproduce, environment details, and expected vs actual behavior.
